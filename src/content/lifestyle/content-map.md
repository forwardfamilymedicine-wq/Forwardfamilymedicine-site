# Lifestyle Medicine Knowledge Library — Content Map

## Architecture Overview

The Knowledge Library uses a pillar + cluster model:
- **7 pillar pages** (2,500–4,000 words each): comprehensive topic hubs that rank for broad queries and link out to all cluster articles in their category
- **30 cluster articles** (1,000–1,500 words each): targeted articles addressing specific questions within each pillar topic; they link back to their pillar and to related pillars

Every page links to `/membership` and `/become-a-patient` via a "How Forward Family Medicine Can Help" section.

## Internal Linking Rules

### Cluster → Pillar (required)
Every cluster article must link back to its parent pillar page. This is the primary internal linking requirement.

### Cross-Pillar Links (encouraged)
Where topics genuinely overlap, cluster articles should link to related pillar pages:

| If your article covers... | Link to... |
|---------------------------|-----------|
| Blood sugar / insulin resistance | `/lifestyle/metabolic-health` |
| Heart disease / cholesterol | `/lifestyle/cardiovascular-prevention` |
| Diet patterns | `/lifestyle/nutrition` |
| Exercise / movement | `/lifestyle/physical-activity` |
| Sleep quality | `/lifestyle/sleep-hormones` |
| Stress / cortisol | `/lifestyle/stress-mental-health` |
| Lifestyle medicine philosophy | `/lifestyle/lifestyle-medicine` |

### Cluster → Cluster (selectively)
Cluster articles can link to closely related cluster articles in the same or adjacent pillar:

**High-value cross-cluster links:**
- `how-to-reverse-prediabetes` → `diet-strategies-blood-sugar` + `getting-started-exercise`
- `sleep-and-metabolic-health` → `what-is-insulin-resistance` + `sleep-and-hormone-regulation`
- `lower-cholesterol-naturally` → `mediterranean-diet-evidence` + `understanding-apob-ldl`
- `blood-pressure-lifestyle` → `dash-diet` context → `anti-inflammatory-diet`
- `zone-2-training` → `hiit-vs-steady-state` + `exercise-for-longevity`
- `stress-physiology-chronic-disease` → `sleep-and-hormone-regulation`

### Pillar → All Clusters (required)
Each pillar page must link to all cluster articles in its category. This creates the hub-and-spoke structure that concentrates authority in the pillar.

### All Pages → Conversion (required)
Every page must link to `/membership` and `/become-a-patient` via the "How Forward Family Medicine Can Help" section.

## Astro Content Collection Configuration

Add `lifestyle` collection to `src/content/config.ts`:

```typescript
import { z, defineCollection } from 'astro:content';

const lifestyleCollection = defineCollection({
  type: 'content',
  schema: z.object({
    title: z.string(),
    description: z.string(),
    keywords: z.array(z.string()),
    canonical: z.string(),
    category: z.string().default('lifestyle'),
    date: z.string(),
    author: z.string(),
    schemaType: z.string().default('Article'),
    references: z.array(z.string()).optional(),
    linksTo: z.array(z.string()).optional(),
    draft: z.boolean().default(false),
  }),
});

export const collections = {
  // ... existing collections
  lifestyle: lifestyleCollection,
};
```

## Routing in Astro

Recommended file structure for routing:

```
src/pages/lifestyle/
  [category]/
    index.astro          → renders pillar pages (lifestyle-medicine/index.md, etc.)
    [slug].astro         → renders cluster articles
```

Or use Astro's content collection routing:

```typescript
// src/pages/lifestyle/[...slug].astro
import { getCollection } from 'astro:content';

export async function getStaticPaths() {
  const entries = await getCollection('lifestyle');
  return entries.map(entry => ({
    params: { slug: entry.slug },
    props: { entry },
  }));
}
```

## Schema Markup

All pages use `schemaType: "Article"`. The SEO component should render:

```json
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "[title]",
  "author": {
    "@type": "MedicalOrganization",
    "name": "Forward Family Medicine",
    "url": "https://forwardfamilymedicine.com"
  },
  "publisher": {
    "@type": "MedicalOrganization",
    "name": "Forward Family Medicine"
  },
  "datePublished": "[date]",
  "mainEntityOfPage": "[canonical]"
}
```

Pages with FAQ sections should also include `FAQPage` schema (see `/src/content/seo/faq-jsonld-snippets.md`).

## AI Optimization Notes

Every article is structured to be extracted by AI systems (Google SGE, ChatGPT, Claude, Perplexity):

1. **Quick answer** in bold under every H1 — provides a one-sentence extractable answer
2. **FAQ sections** — map directly to FAQPage JSON-LD schema and AI question-matching
3. **H2 headings as questions** where applicable — "How to Lower Blood Pressure Naturally" is more extractable than "Interventions"
4. **References section** — signals E-E-A-T; authoritative citations increase AI confidence in extracting content
5. **Consistent structure** — AI systems prefer predictable document structure

## Editorial Standards

- All medical claims must be attributed to peer-reviewed research, ACLM guidelines, CDC data, or major clinical organizations (AHA, ADA, ACC)
- Do not invent statistics; always cite the source study
- Maintain physician-authored voice (warm, evidence-led, prescriptive)
- Update references when major guideline changes occur (e.g., ADA Standards of Medical Care updated annually)
- "How Forward Family Medicine Can Help" sections must use "If you'd like individual guidance..." framing — not promotional
- All quantitative claims should specify whether they are relative risk reductions, absolute risk reductions, or effect sizes — do not conflate these
